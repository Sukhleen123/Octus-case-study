# Financial Multi-Agent Intelligence System

## Executive Summary

This repo implements the **data ingestion layer** of a Financial Multi-Agent Intelligence System that answers natural-language questions over:

- **Octus unstructured documents**: 15 companies, 105 earnings call transcripts, 137 SEC filings (10-K / 10-Q) with full HTML content
- **SimFin structured financial statements**: income, balance sheet, cash flow — batch download or live v3 API

The system features:
- **UI-first, zero-CLI workflow**: run `streamlit run apps/streamlit_app.py` — bootstrap is automatic
- **Idempotent bootstrap**: `ensure_ready(settings)` computes a fingerprint of raw files + settings, skips rebuild if nothing changed
- **Auto-trigger on settings change**: changing any settings field (chunk size, retriever, mapping mode, etc.) automatically triggers a rebuild
- **Mock-mode demo**: runs with `embedding_provider=mock` and FAISS fallback — no API keys required
- **A/B experiment grid**: `configs/experiments.yaml` defines a grid over chunker / retriever / periodicity / vectorstore

## Quick Start

### 1. Install dependencies

```bash
pip install -e ".[dev]"
```

### 2. Configure environment

```bash
cp .env.example .env
# Edit .env — minimum required for full functionality:
#   SIMFIN_API_KEY=...
#   PINECONE_API_KEY=... (optional; FAISS fallback used if absent)
```

### 3. Run the Streamlit UI (recommended)

```bash
streamlit run apps/streamlit_app.py
```

On first run, bootstrap will:
1. Parse and normalize Octus JSON/HTML → `data/processed/octus/documents.parquet`
2. Build heading + token chunk tables
3. Match Octus companies to SimFin tickers (rapidfuzz)
4. Download SimFin financial statements (if `SIMFIN_API_KEY` set)
5. Embed all chunks with MockEmbedder (or OpenAI if configured)
6. Index into FAISS (or Pinecone if key set)

Subsequent runs reuse existing artifacts (fingerprint match → instant startup).

### 4. Run the Chainlit UI (alternative)

```bash
chainlit run apps/chainlit_app.py
```

### 5. Run tests

```bash
pytest tests/ -v
```

### 6. Explore data in notebooks

```bash
jupyter lab notebooks/
```

## Key Settings

All settings live in `configs/app.yaml` and can be overridden via environment variables:

| Setting | Default | Purpose |
|---------|---------|---------|
| `TABLE_FORMAT` | `parquet` | `parquet` or `csv` for all output tables |
| `EMBEDDING_PROVIDER` | `mock` | `mock` (no key needed) or `openai` |
| `VECTORSTORE_PROVIDER` | `auto` | `auto` (Pinecone if key set, else FAISS) |
| `SIMFIN_MODE` | `batch` | `batch` or `realtime` |
| `SIMFIN_PERIODICITY` | `annual` | `annual`, `quarterly`, or `both` |
| `MAPPING_MODE` | `auto_matched` | Which Octus→SimFin mappings to use |
| `RETRIEVER_ID` | `dense` | `dense` or `dense_mmr` |
| `CHUNK_SIZE` | `512` | Tokens per window (token chunker) |
